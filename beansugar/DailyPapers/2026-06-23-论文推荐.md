### 1. SecFSM: Knowledge Graph-Guided LLMs for Generating Secure Verilog Code of Finite State Machine in SoCs

- **作者**: Ziteng Hu, Yingjie Xia, Jiachi Chen, Kuang Li, Yao Wan
- **机构**: Hangzhou Dianzi University, Zhejiang University, Central South University, Huazhong University of Science and Technology
- **链接**: [IEEE Xplore](https://doi.org/10.1109/TDSC.2026.3679566)
- **来源**: 📄 arXiv 关键词检索
- 📒 **笔记**: [[SecFSM]]

- **核心方法**: 输入是自然语言描述的 FSM 规格说明，输出是经过安全加固的 Verilog FSM 代码。方法分两阶段：(1) [[SecFSM|Structure-aware FSM Graph Analysis]] 从规格中提取状态转移图结构，识别结构级漏洞（如 Dead State）；(2) [[FSKG|Knowledge-guided Vulnerability Confirmation]] 从手工构建的 FSM Security Knowledge Graph 中检索漏洞相关的安全知识（CWE 条目、Good Example、修复建议），构造成安全 prompt 约束 LLM 输出。底层 LLM 可以是 GPT-4o、Claude 3.5 或 DeepSeek-R1，框架本身是 model-agnostic 的。与 [[RAG]] 的关键区别在于：RAG 只做粗粒度检索增强，SecFSM 在检索前先做结构分析确定"查什么"，检索后做知识组合构造"怎么修"。
- **对比方法/Baselines**: Base（纯 LLM 直出）、RAG（TinyRAG 检索增强）。基底模型涉及 [[GPT-4o]]、[[Claude 3.5]]、[[DeepSeek-R1]]。借鉴/对比的工作包括 [[VerilogCoder]]、[[RTLCoder]]、[[ChipNeMo]]、[[AutoGen]]（用于 security prompt construction）、[[VerilogEval]]（benchmark）。FSM 安全检测借鉴了 [[ARC-FSM]] 的 netlist-level 规则检查理念。
- **借鉴意义**: 对做 embodied AI 的人直接借鉴意义有限，但方法论值得注意——"先分析结构再检索知识、再组合 prompt"这个范式可以迁移到其他需要领域知识约束的 LLM 代码生成场景（比如你的 robot control policy 生成如果有 safety constraint）。[[FSKG]] 的手工构建方式（top-down ontology + manual cleaning + entity alignment）也值得参考。
- **锐评**: 这是一篇工程上认真、实验设计相当完整的论文。优点具体说：消融实验拆了 Stage I 和 Stage II 各自的贡献（去 Stage I 安全从 75% 跌到 46%、去 Stage II 跌到 69%）；做了 inter-annotator agreement（Cohen's Kappa 0.951）；有 failure study（RQ4）追踪 pipeline 各阶段失败率；有 case study 展示完整流程。DeepSeek-R1 达到 48/52 安全通过率，比 Base（19/52）和 RAG（19/52）有实质性提升。但问题也很明显：(1) [[FSKG]] 是手工构建的——52 个测试用例的规模在真实芯片设计中杯水车薪，FSM 的安全漏洞类型是有限的所以能手工覆盖，但扩展到其他硬件模块（crypto controller、bus fabric）时手工维护知识图谱不可持续，作者在 future work 里也承认了这一点；(2) 方法对非标准状态描述格式（如 q2afsm 用例）会提取错误信息导致误判；(3) 整个 pipeline 引入了额外的延迟和计算开销——Structure-aware Graph Analysis + Cypher 查询 + LLM 多轮交互，这对于需要快速迭代的 EDA workflow 是个实际负担，作者没给出延迟数据；(4) prompt 构造的可靠性不足——DeepSeek-R1 的 secure prompt 生成准确率只有 85%，意味着还有 15% 的概率 prompt 本身就写错了。总体而言：方向对、方法务实、实验完整，但规模问题限制了实用性。能在 IEEE TDSC 上发表说明同行认可度是有的。🔥
