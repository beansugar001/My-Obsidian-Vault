# 科研项目管理方法论

> 基于 `植物病害识别系统` 项目的 MEMORY.md 总结而来
> 目的：规范未来项目的记录方式，提高工作效率和可追溯性

---

## 一、文件结构规范

### 1.1 顶层结构

```markdown
# 用户偏好

# 当前项目：[项目名称]

## 项目配置

## 当前进度

## 历史记录

### 当前状态

## 项目子模块

### 学习内容

```

### 1.2 用户偏好部分

**目的**：记录用户的个性化设置，避免重复询问

**必填项**：
- **语言**：始终使用XX回答
- **包安装路径**：如果使用自定义包路径
- **其他偏好**：代码风格、工具选择等

**示例**：
```markdown
# 用户偏好

- **语言**: 始终使用中文回答
- **包安装规则**: 所有Python包必须安装到D盘 `D:\pythonpackage\packages`
- **代码风格**: 使用类型注解、添加docstring
```

## 二、项目配置规范

### 2.1 基本信息

**必填项**：
- **路径**：项目根目录的绝对路径
- **环境**：conda环境名、CUDA版本、硬件配置
- **Python路径**：完整路径，标注注意事项
- **PYTHONPATH**：如果有自定义路径

**示例**：
```markdown
## 项目配置
- **路径**: `D:\毕业论文\代码部分`
- **环境**: thesis conda环境 (CUDA 12.1, GTX 1650 4GB)
- **Python路径**: `C:/Users/24796/.conda/envs/thesis/python.exe` ⚠️ 必须使用conda环境，不能用系统Python
- **PYTHONPATH**: `D:\pythonpackage\packages`
```

### 2.2 命令模板

**目的**：避免重复输入长命令

**格式**：
```markdown
## 包安装命令模板
```bash
[完整命令]
```
```

## 三、进度记录规范

### 3.1 进度列表格式

**格式**：
```markdown
## 当前进度
1. ✅ 已完成任务 (关键信息)
2. ⏳ 进行中任务 (预计时间)
3. ⚠️ 遇到的问题 (问题描述)
```

**状态符号**：
- `✅` 已完成
- `⏳` 进行中
- `⚠️` 有问题
- `❌` 失败/废弃

### 3.2 进度项编写规则

**原则**：简洁明了，包含关键信息

**示例**：
```markdown
✅ 好的进度记录：
- ✅ 训练完成 (50 epochs, 显存占用11%)
- ✅ 修复Windows共享内存错误 (num_workers=0)

❌ 不好的进度记录：
- ✅ 训练完成
- ✅ 修复bug
```

## 四、问题记录规范

### 4.1 顶层问题记录

**格式**：
```markdown
## [日期] 主题

### N. [问题标题] ✅/❌
**问题**: [简明描述问题]

**解决方案**:
1. 步骤一
2. 步骤二

**修改文件**:
- `绝对路径1` - 修改说明
- `绝对路径2` - 修改说明
```

### 4.2 问题标题规则

**格式**：`动词 + 对象 + 目的/结果`

**示例**：
- ✅ 好的标题：
  - "修复ROC曲线模型加载错误"
  - "优化数据集样本展示图字体"
  - "补充第二章损失函数理由"

- ❌ 不好的标题：
  - "ROC曲线问题"
  - "字体优化"
  - "补充内容"

### 4.3 问题描述规范

**必须包含**：
1. **现象**：看到了什么问题
2. **影响**：造成了什么后果
3. **原因**：为什么会出现（如果已知）

**示例**：
```markdown
**问题**:
1. ROC曲线AUC只有0.47左右（接近随机预测0.5）
2. 模型预测接近随机状态
3. 使用 `strict=False` 加载导致参数未正确匹配
```

### 4.4 解决方案规范

**原则**：步骤清晰，可复现

**格式**：
```markdown
**解决方案**:

#### 1. 小标题
- 修改前：[代码/配置]
- 修改后：[代码/配置]
- 说明：[为什么这样改]

#### 2. 小标题
- 步骤1
- 步骤2
```

## 五、文件路径记录规范 ⭐

### 5.1 绝对路径原则

**原则**：所有文件路径必须使用从盘符开始的完整绝对路径

**格式**：
```markdown
# ✅ 正确
- `D:\毕业论文\代码部分\Code\tools\generate_roc_curves.ipynb`

# ❌ 错误
- `Code/tools/generate_roc_curves.ipynb`
- `generate_roc_curves.ipynb`
- ./tools/generate_roc_curves.ipynb
```

### 5.2 路径记录场景

**需要记录路径的场景**：
1. 修改的文件
2. 新建的文件
3. 读取的文件
4. 输出文件
5. 配置文件
6. 数据文件
7. 模型文件

**示例**：
```markdown
**修改文件**:
- `D:\毕业论文\03_Overleaf上传模板_无需解压缩_zjouthesis_Mod\data\chap02.tex` - 补充损失函数理由

**相关文件路径**:
- ROC曲线生成: `D:\毕业论文\代码部分\Code\tools\generate_roc_curves.ipynb`
- 模型文件: `D:\毕业论文\代码部分\models\best_swin_grabcut.pth`
- 数据集: `D:\毕业论文\代码部分\datasets\images_80_20\val`
- 输出文件: `D:\毕业论文\代码部分\visualization_results\roc_curves.pdf`
```

### 5.3 路径表格化

当有多个相关文件时，使用表格记录：

**示例**：
```markdown
| 文件类型 | 完整路径 |
|---------|----------|
| ROC曲线生成 | `D:\毕业论文\代码部分\Code\tools\generate_roc_curves.ipynb` |
| 键名检查工具 | `D:\毕业论文\代码部分\Code\tools\check_checkpoint_keys.py` |
| 正确的模型 | `D:\毕业论文\代码部分\models\best_swin_grabcut.pth` ⭐ |
| 错误的模型 | `D:\毕业论文\代码部分\models\best_swin_multiclass_10epochs.pth` ❌ |
```

## 八、经验教训记录规范

### 8.1 经验总结格式

```markdown
**经验教训**:
1. **[类别]**
   - 问题：[描述]
   - 解决：[方法]
   - 预防：[下次如何避免]
```

### 8.2 常见经验类别

- 模型加载
- 数据处理
- 环境配置
- 性能优化
- 调试技巧
- 文件管理

**示例**：
```markdown
**经验教训**:
1. **模型加载**
   - 问题：checkpoint键名与模型键名不匹配
   - 解决：使用 `strict=True` 确保完全匹配
   - 预防：加载前先检查键名格式，使用诊断工具

2. **文件路径**
   - 问题：相对路径在不同环境下失效
   - 解决：统一使用绝对路径
   - 预防：所有文件路径记录时使用完整绝对路径
```

## 十、最佳实践

### 10.1 记录时机

**即时记录原则**：
- ✅ 问题解决后立即记录
- ✅ 重要修改后立即记录
- ✅ 实验完成后立即记录
- ❌ 不要等到第二天

### 10.2 记录详略

**详略得当**：
- ✅ 关键步骤详细记录
- ✅ 踩坑经历详细记录
- ✅ 创新点详细记录
- ⚠️ 常规操作简要记录
- ⚠️ 已知内容省略

### 10.3 可追溯性

**确保可追溯**：
- 每个修改都有文件路径
- 每个问题都有解决方案
- 每个实验都有结果记录
- 每个决策都有理由说明

### 10.4 搜索友好

**便于搜索**：
- 使用关键词作为标题
- 使用表格汇总信息
- 使用代码块标记代码
- 使用绝对路径便于查找

## 十一、维护建议

### 11.1 定期整理

**建议**：
- 每周整理一次当前状态
- 每月总结一次关键进展
- 项目结束后归档

### 11.2 版本控制

**建议**：
- MEMORY.md本身纳入版本控制
- 重要节点创建快照
- 保留历史版本以便回溯

### 11.3 分层记录

**建议**：
- MEMORY.md：详细记录
- README.md：项目概览
- 各模块文档：深入细节

## 结语

本方法论基于实际项目经验总结，核心原则是：

1. **绝对路径**：避免路径混乱
2. **即时记录**：避免遗漏
3. **详略得当**：提高可读性
4. **可追溯**：确保可复现
5. **持续改进**：不断完善

遵循本方法论，可以：
- 快速定位问题
- 高率复现结果
- 积累项目经验
- 提升科研效率
