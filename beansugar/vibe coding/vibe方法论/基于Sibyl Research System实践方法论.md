## 🎯 核心理念

### 什么是Vibe Research？

**Vibe Research = 直觉 + 探索 + AI加速**

一种新的研究范式，让AI Agent承担重复性、技术性的工作，人类专注于创意方向和直觉判断。

### 三大原则

1. **直觉 > 数据**
   - 跟随你的vibe和直觉
   - 数据是用来验证vibe的，不是替代vibe
   - 不要因为"没有数据"就不敢尝试

2. **快速迭代 > 完美主义**
   - 不要追求第一次就完美
   - 让AI Agent快速试错
   - 失败也是vibe的一部分

3. **AI做脏活，我做创意**
   - 文献调研：AI
   - 代码编写：AI
   - 实验运行：AI
   - 论文初稿：AI
   - 创意方向：我
   - 最终把控：我

## 💻 环境部署

### 本地部署 vs 云服务

#### 成本对比
| 方式 | 成本/月 | 成本/年 | 适用场景 |
|------|---------|---------|----------|
| **本地部署** | 0元 | 0元 | 个人研究、vibe自由 |
| 云服务器 | 670-1,159元 | 8,040-13,908元 | 团队协作、需要更多资源 |

**节省**: 本地部署年节省8,040-13,908元，可以用来vibe更久！

### 最小硬件要求
- **GPU**: NVIDIA GTX 1650 (4GB) 或更好
- **CPU**: 现代多核处理器
- **内存**: 16GB+
- **存储**: 50GB+ 可用空间
- **操作系统**: Windows/Linux/macOS

### 软件环境
```bash
# Python环境
Python 3.12+

# Claude Code CLI
claude --version  # 需要 v2.1.0+

# API Key
ANTHROPIC_API_KEY  # 可使用官方或代理（如zai）
```

### 部署步骤（Windows）

#### 1. 克隆仓库
```bash
git clone https://github.com/Sibyl-Research-Team/sibyl-research-system.git
cd sibyl-research-system
```

#### 2. 创建虚拟环境
```bash
# 创建D盘环境（避免C盘空间不足）
python -m venv D:\conda-envs\sibyl
```

#### 3. 安装依赖
```bash
D:\conda-envs\sibyl\python.exe -m pip install -e .
```

#### 4. 配置API Key
```bash
# 设置环境变量
setx ANTHROPIC_API_KEY "your-api-key"

# 或使用代理（如zai）
setx ANTHROPIC_API_KEY "zai-proxy-key"
```

#### 5. 创建配置文件
```yaml
# config.yaml
compute_backend: local        # 使用本地GPU
local_conda_env_name: sibyl   # 虚拟环境名称
language: zh                  # 中文界面
workspaces_dir: D:\sibyl-workspaces
```

#### 6. Windows兼容性修复
需要修复以下文件（已在新版本中修复）：
- fcntl模块（6个文件）
- 虚拟环境路径（3个文件）
- 符号链接（2个文件）

### 阶段2: 项目启动（1天）

#### 编写研究规格书（spec.md）
```markdown
# 项目: [项目名称]

## 研究主题
[一句话描述]

## 背景与动机
[为什么这个问题重要]

## 初始想法
- 方向1：...
- 方向2：...
- 方向3：...

## 关键约束
- 硬件限制
- 时间限制
- 数据限制

## 目标产出
- 论文/技术报告

- 最佳实践文档
```

#### 配置项目参数
```yaml
# config.yaml
max_gpus: 4                    # GPU数量
pilot_samples: 100             # pilot样本数
debate_rounds: 2               # 辩论轮数
writing_mode: parallel         # 写作模式
language: zh                   # 语言
```

### 阶段3: 自动化研究（数天到数周）

#### 研究循环（19阶段）

**阶段1-5: 文献调研与创意生成**
1. 文献检索 (arXiv + Web) → 1-2小时
2. 创意辩论 (6个Agent) → 30分钟-1小时
3. 实验规划 → 30分钟
4. 先导实验 → 1-2小时
5. 完整实验 (GPU并行) → 数小时

**阶段6-10: 结果分析与论文写作**
6. 结果辩论 (6个Agent) → 1小时
7. 论文大纲 → 30分钟
8. 章节写作 (6个Agent并行) → 2-4小时
9. 交叉评审 (6个Agent) → 1-2小时
10. 整合编辑 → 1小时

**阶段11-15: 质量检查与迭代**
11. 终审 (NeurIPS级别) → 1小时
12. 修改 → 1-2小时
13. 质量门控 → 自动判断
14. 转向或继续 → 自动决策
15. 重复实验（如需要）

**阶段16-19: 最终产出**
16. 论文定稿 (LaTeX) → 1-2小时
17. 编译PDF → 30分钟
18. 提交材料准备 → 1小时
19. 项目归档 → 自动完成

### 阶段4: 结果评估（持续）

#### 质量指标
- **论文质量**: 1-10分自动评分
- **实验完整性**: 是否完成所有实验
- **创新性**: 是否有新颖发现
- **可复现性**: 实验是否可复现

#### 迭代决策
- **继续迭代**: 质量未达标，有改进空间
- **转向新想法**: 当前方向遇阻
- **终止并产出**: 质量达标，准备投稿

## 💡 核心技巧

### 1. 选择合适的研究主题

**好的Vibe Research主题**:
- ✅ 基于你的直觉和兴趣
- ✅ 有明确的应用场景
- ✅ 可在现有硬件上完成
- ✅ 有一定的创新空间

**避免的主题**:
- ❌ 需要大规模训练数据
- ❌ 需要昂贵的云计算资源
- ❌ 完全脱离你的知识背景
- ❌ 过于理论化，缺乏实践

### 2. 编写有效的spec.md

**关键要素**:
```markdown
## 研究主题
- 简洁明确（一句话）
- 包含关键词

## 初始想法
- 至少3个不同方向
- 每个方向有具体方案

## 关键约束
- 明确硬件限制
- 明确时间限制
- 明确数据限制

## 目标产出
- Workshop论文（现实）
- 开源代码（加分项）
```

### 3. 监控和调整

**定期检查**（每30分钟）:
- Web UI项目状态
- 错误日志
- 实验进度

**及时干预**:
- 发现方向错误 → 暂停并调整
- 资源不足 → 调整实验规模
- 遇到错误 → 查看日志并修复

### 4. 结果评估

**质量门控**:
- 7分以上：考虑投稿
- 5-7分：继续迭代
- 5分以下：转向新想法

**人工审查**:
- 论文逻辑是否通顺
- 实验设计是否合理
- 创新点是否突出

## ⚠️ 注意事项

### 安全提醒

**`--dangerously-skip-permissions`标志**:
- 允许Claude不经确认执行命令
- 每轮迭代涉及数百次操作
- ⚠️ 仅在专用研究机器上使用
- ⚠️ 不要在存放敏感数据的机器上使用

### Windows兼容性

**已知问题**:
- fcntl模块不支持 → 已修复
- 符号链接需要权限 → 已修复
- 路径格式差异 → 已修复

**解决方案**:
- 使用最新版本（已包含修复）
- 或参考修复补丁

### 性能优化

**本地GPU限制**:
- 使用轻量级模型
- 优先推理任务
- 避免大规模训练

**加速技巧**:
- 减少debate_rounds
- 减少pilot_samples
- 使用parallel写作模式

**不是让AI替代人类，而是让AI放大人类的能力**

- 🎨 人类提供：创意、直觉、方向
