# Content Reviewer Skill

对静态内容（论文、图片、表格、文档等）进行多维度专业审查，发现错误并生成详细的错误-修正对照报告。

## Skill Location

**Skill Location**: `.claude/skills/content-reviewer/` 或用户自定义 skills 目录

> **注意**：本 skill 专注于静态内容审查，代码 PR 审查请使用内置的 `review` skill。

## 核心方法论：渐进式审查法

本skill提供两种审查模式，可根据需求选择：

### 审查模式对比

| 模式 | 轮次 | 适用场景 | 耗时 | 准确度 |
|------|------|---------|------|--------|
| 快速模式 | 3轮 | 日常检查、初稿审查 | 低 | 中等 |
| 深度模式 | 10轮 | 学术发表、正式交付 | 高 | 高 |

### 快速模式（3轮）

| 轮次 | 审查维度 | 核心关注点 | 适用内容类型 |
|------|---------|-----------|-------------|
| 第1轮 | 整体理解与逻辑 | 主题、结构、逻辑连贯性、完整性 | 全部 |
| 第2轮 | 内容准确性 | 术语、公式、数据、数值、文字拼写 | 全部 |
| 第3轮 | 格式与规范 | 排版、样式、格式规范、技术准确性 | 全部 |

### 深度模式（10轮）

| 轮次 | 审查维度 | 核心关注点 | 适用内容类型 |
|------|---------|-----------|-------------|
| 第1轮 | 整体内容理解 | 主题、主要元素、整体结构、用途 | 全部 |
| 第2轮 | 结构与逻辑 | 结构合理性、逻辑连贯性、层次关系、完整性 | 全部 |
| 第3轮 | 文字与拼写 | 拼写错误、语法错误、大小写、标点、文字截断 | 全部 |
| 第4轮 | 术语与公式 | 专业术语准确性、数学公式、变量名、符号 | 论文/图片/技术文档 |
| 第5轮 | 数据与数值 | 数值正确性、维度一致性、计算验证、单位 | 论文/图片/表格 |
| 第6轮 | 逻辑一致性 | 因果关系、前后矛盾、自洽性、与标准/规范对比 | 全部 |
| 第7轮 | 排版与对齐 | 对齐、间距、字体统一、元素重叠 | 图片/表格/文档 |
| 第8轮 | 格式与样式 | 标注位置、颜色一致性、样式统一、视觉层次 | 图片/表格/文档 |
| 第9轮 | 技术准确性 | 与行业标准/原论文/规范对比、遗漏组件 | 论文/图片/技术文档 |
| 第10轮 | 综合复查 | 之前遗漏、边界情况、整体评估 | 全部 |

### 审查流程

```
输入内容 → 识别内容类型 → 加载对应审查清单 → 执行10轮审查 → 汇总问题 → 生成报告
```

## 内容类型识别与审查策略

### 1. 图片审查（流程图/架构图/示意图/海报等）

**输入方式**: 使用 MCP Vision 工具（`mcp__4_5v_mcp__analyze_image`）对图片进行分析

**审查示例**:
```python
# 使用 MCP Vision 工具
prompt = """请对这张图进行第1轮审查，聚焦整体内容理解维度。
审查重点：主题、主要元素、整体结构、用途
请逐一列出你发现的所有问题，对每个问题说明：
1. 问题位置（具体在哪个部分）
2. 问题描述（具体什么错误）
3. 严重程度（严重/中等/轻微）
4. 修正建议（如何修改，给出具体修正内容）"""
```

**审查重点**:
- 箭头方向与连接关系
- 维度标注与数据一致性
- 文字拼写与术语准确性
- 排版对齐与视觉统一性
- 技术架构与标准/原论文的一致性

### 2. 论文审查（学术论文/技术报告等）

**输入方式**: 使用 Web Reader MCP 解析在线 PDF/DOCX

**审查重点**:
- 摘要与正文的一致性
- 公式推导的正确性
- 引用格式与完整性
- 实验设置的可复现性
- 图表与正文描述的一致性
- 结论与数据的支撑关系

### 3. 表格审查（数据表/统计表/对比表等）

**输入方式**: 使用 MCP Vision 工具分析表格图片

**审查重点**:
- 数据计算的正确性（行列合计、百分比等）
- 表头与数据的对齐关系
- 单位的一致性
- 缺失值的标注
- 数据与文字描述的一致性

### 4. 通用文档审查

**审查重点**（详见 `references/general-checklist.md`）:
- 文字拼写与语法
- 逻辑连贯性
- 格式规范性
- 信息完整性

## 执行步骤

### Step 1: 确认文件与内容类型

1. 确认待审查文件的路径和格式
2. 识别内容类型（图片/论文/表格/文档）
3. 选择审查模式（快速/深度）
4. 加载对应的审查清单

### Step 2: 执行审查

对每一轮审查：
1. 根据审查维度构造专门的审查提示词
2. 使用对应工具执行审查
3. 记录发现的所有问题
4. 为每个问题标注严重程度（严重/中等/轻微）

**审查提示词模板**:
```
请对这份{内容类型}进行第{N}轮审查，聚焦{审查维度}维度。
审查重点：{具体检查项}
请逐一列出你发现的所有问题，对每个问题说明：
1. 问题位置（具体在哪个部分）
2. 问题描述（具体什么错误）
3. 严重程度（严重/中等/轻微）
4. 修正建议（如何修改，给出具体修正内容）
```

### Step 3: 问题去重与分类

1. 合并各轮审查中重复发现的问题
2. 按严重程度分类：严重 > 中等 > 轻微
3. 按问题类型分组：技术性错误 > 内容性错误 > 排版格式问题

### Step 4: 生成报告

生成 Markdown 或 docx 格式的审查报告。

**报告核心要求**:
- **问题-修正对照表格**：每个问题必须配有详细的修正方案，以表格形式并排展示
- 修正方案必须具体可操作，给出明确的修改内容（而非笼统建议）
- 按严重程度排序

**报告结构**:
1. 封面（审查对象、审查日期、审查方法）
2. 审查概述（内容类型、审查模式、问题统计）
3. 问题汇总表（详见下方表格定义）
4. 分维度详细审查结果
5. 审查方法说明
6. 修正优先级建议

## 问题严重程度定义

| 级别 | 定义 | 示例 |
|------|------|------|
| 严重 | 技术性错误，影响内容正确性，可能误导读者 | 维度标注错误、公式错误、数据计算错误 |
| 中等 | 影响可读性或专业性，但不影响内容正确性 | 标注位置不统一、缺少关键标注、逻辑表述不清 |
| 轻微 | 影响美观度或一致性，不影响内容理解 | 字体大小差异、颜色微小差异、箭头粗细不一 |

## 报告生成

使用 Write 工具生成 Markdown 报告，或使用 docx skill 生成 docx 格式报告。

报告中的**问题-修正对照表格**是核心输出：

| 列名 | 说明 |
|------|------|
| 编号 | 问题序号 |
| 严重程度 | 严重/中等/轻微 |
| 问题位置 | 问题出现在内容的哪个具体位置 |
| 问题描述 | 当前内容中存在的具体错误 |
| 修正方案 | 具体的修改操作步骤（如"将X改为Y"、"删除Z"） |
| 修正后内容 | 修改后应呈现的完整、正确内容 |

**关键要求**：
- "修正方案"说明**如何修改**（操作步骤）
- "修正后内容"给出**修改后的结果**（最终内容）
- 对于数值错误，必须给出计算过程和最终正确值
- 对于文字错误，必须给出完整的修正后文字

## 注意事项

1. **模式选择**：日常检查使用快速模式（3轮），正式发表使用深度模式（10轮）
2. **审查独立性**：每轮审查应独立进行，不因前几轮的发现而跳过检查项
3. **问题记录**：即使同一问题在多轮中被重复发现，也应记录首次发现的轮次
4. **修正可操作性**：修正方案必须具体到可直接执行，而非笼统建议
5. **语言一致性**：报告语言应与被审查内容的语言一致
